Sitemap 应该提交哪些页面

sitemap 只收录高质量公开页面，不要把所有文件都塞进去

Sitemap 的作用

Sitemap 告诉搜索引擎你的站点有哪些页面值得抓取和收录。它不是保证收录的手段，而是一个发现机制。

应该放入 Sitemap 的页面

  • 所有希望被收录的公开页面
  • 返回 200 状态码的页面
  • 有 canonical 自指的页面
  • 没有 noindex 的页面
  • 有实质内容的页面

不应该放入 Sitemap 的内容

  • 非 HTML 文件（PDF、图片用媒体 sitemap）
  • 带参数的重复 URL
  • 已设置 noindex 的页面
  • 301/302 重定向的源 URL
  • 返回 4xx/5xx 的 URL
  • 低质量标签页、空分类页
  • 内部搜索结果页
  • 需要登录的页面

首批提交规则

  1. 新站首次提交不超过 1000 个 URL
  2. 优先提交最重要、内容最充实的页面
  3. 确保每个 URL 都能正常访问（200 状态码）
  4. 使用 lastmod 标注真实的最后修改时间
  5. 提交后在 GSC 中监控索引覆盖率变化
  6. 根据收录情况逐步增加 URL

Sitemap 格式示例

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/page</loc>
    <lastmod>2024-01-15</lastmod>
  </url>
</urlset>
