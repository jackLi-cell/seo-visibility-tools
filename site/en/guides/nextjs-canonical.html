Common Next.js Canonical Configuration Errors

Correct metadata and canonical configuration in Next.js App Router

Next.js App Router 中的 Canonical

Next.js 13+ App Router Use metadata API 来设置页面的 canonical URL。配置不当是导致收录问题的Common Causes。

Common Causes

  • 忘记在 generateMetadata 中设置 canonical
  • Use process.env.NEXT_PUBLIC_URL 但环境变量未设置
  • canonical 中包含查询参数或 hash
  • 开发环境用 localhost，生产环境忘记改
  • 动态路由中 canonical 没有包含实际的 slug

Troubleshooting Steps

  1. 查看部署后页面源码，搜索 rel="canonical" 确认输出
  2. Check canonical URL 是否为完整的绝对路径（含 https://）
  3. 确认动态路由页面的 canonical 包含正确的 slug 参数
  4. Check metadataBase 是否在 layout.tsx 中正确设置
  5. 验证环境变量 NEXT_PUBLIC_URL 在生产环境中有值
  6. 对比开发环境和生产环境的 canonical 输出
  7. Check是否有中间件或插件覆盖了 canonical 设置

静态 Metadata 配置

// app/blog/[slug]/page.tsx
export const metadata = {
  alternates: {
    canonical: 'https://example.com/blog/my-post',
  },
};

动态 generateMetadata

// app/blog/[slug]/page.tsx
export async function generateMetadata({ params }) {
  const slug = params.slug;
  return {
    title: '文章标题',
    alternates: {
      canonical: 'https://example.com/blog/' + slug,
    },
  };
}

Layout 级别的默认配置

// app/layout.tsx
export const metadata = {
  metadataBase: new URL('https://example.com'),
};
// 设置 metadataBase 后，canonical 可以Use相对路径

Fix Methods

  • 在 app/layout.tsx 中设置 metadataBase 为生产域名
  • 为每个动态路由页面实现 generateMetadata
  • 确保环境变量在所有部署环境中正确配置
  • Use Next.js 的 metadata 验证ToolsCheck输出
  • 在 CI/CD 中添加 canonical Check步骤

Verification

部署后查看页面源码，确认 canonical 输出了正确的完整 URL。Use Canonical Checker 验证。
