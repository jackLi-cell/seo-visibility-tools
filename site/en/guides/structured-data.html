Structured Data and FAQ Basic Check

Basic check methods and common errors for JSON-LD structured data

什么是结构化数据

结构化数据是用标准格式（通常是 JSON-LD）向搜索引擎描述页面内容的方式。它帮助搜索引擎理解页面的类型、内容和关系。

Common Causes（结构化数据问题）

  • JSON 格式错误（缺少逗号、引号不匹配）
  • Use了不存在的 schema 类型或属性
  • 结构化数据内容与页面可见内容不一致
  • 缺少必需属性
  • 日期格式不正确（应Use ISO 8601）

Troubleshooting Steps

  1. 查看页面源码，找到所有 script type="application/ld+json" 标签
  2. 将 JSON-LD 内容粘贴到 JSON 验证器中Check格式
  3. Use Google 富媒体搜索结果测试Tools验证 schema 有效性
  4. 对比结构化数据中的内容与页面可见内容是否一致
  5. Check是否缺少必需属性（如 Article 需要 headline、author）
  6. 确认日期格式为 ISO 8601（如 2024-01-15）
  7. 验证 URL 字段Use绝对路径而非相对路径

JSON-LD 基础

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "页面标题",
  "description": "页面描述",
  "dateModified": "2024-01-15",
  "author": {
    "@type": "Person",
    "name": "作者名"
  }
}
</script>

FAQPage 结构化数据

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "问题1？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "答案1"
      }
    }
  ]
}
</script>

WebApplication 结构化数据

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebApplication",
  "name": "Tools名称",
  "description": "Tools描述",
  "applicationCategory": "SEO Tool",
  "operatingSystem": "Web Browser",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "USD"
  }
}
</script>

Fix Methods

  • Use在线 JSON 验证器修复格式错误
  • 参考 schema.org 官方文档确认属性名称
  • 确保结构化数据与页面可见内容完全一致
  • 添加所有必需属性，补充推荐属性
  • 统一Use ISO 8601 日期格式

Verification

Use HTML Readability Check Tool Check JSON-LD 格式是否有效。也可以Use Google 的富媒体搜索结果测试Tools在线验证。
