How to Check Robots.txt and Noindex

Check if robots.txt, meta robots, and X-Robots-Tag accidentally block pages

Robots.txt Basics

robots.txt is a text file placed in the website root directory that tells crawlers which paths can be crawled and which cannot. Note: robots.txt is only a suggestion, not a mandatory block.

Robots.txt Syntax

# 允许所有爬虫访问所有内容
User-agent: *
Allow: /

# 屏蔽特定目录
User-agent: *
Disallow: /admin/
Disallow: /private/

# 指定 Sitemap 位置
Sitemap: https://example.com/sitemap.xml

Meta Robots Tag

<!-- 不收录此页面 -->
<meta name="robots" content="noindex">

<!-- 不收录且不跟踪链接 -->
<meta name="robots" content="noindex, nofollow">

X-Robots-Tag HTTP Header

X-Robots-Tag: noindex
X-Robots-Tag: noindex, nofollow

Common Accidental Blocking Scenarios

  • robots.txt Disallow: / blocking the entire site
  • noindex tag from test environment carried to production
  • CMS "Discourage search engines" option is checked
  • CDN or server configuration added global X-Robots-Tag
  • Framework adds noindex to certain routes by default

Troubleshooting Steps

  1. Visit /robots.txt to check for over-blocking
  2. View page source and search for "noindex"
  3. Use developer tools to check X-Robots-Tag in response headers
  4. Check CMS backend SEO settings
  5. Confirm CDN configuration has not added extra robots headers

Verification

Use Robots Checker paste your robots.txt or meta tags to check.
