IndexNow Protocol Usage Guide

Proactively notify search engines of content updates via IndexNow protocol to accelerate Bing and Yandex indexing

What is IndexNow

IndexNow 是一个开放协议，允许网站主动通知搜索引擎内容已更新。支持 IndexNow 的搜索引擎包括 Bing、Yandex、Seznam 和 Naver。Google 目前不支持但在评估中。

Common Causes（收录延迟）

  • 依赖搜索引擎自然发现，没有主动通知机制
  • Sitemap 更新频率低，搜索引擎不知道内容已变化
  • 新页面发布后没有任何推送动作
  • 不了解 IndexNow 可以显著加速 Bing 收录

Troubleshooting Steps

  1. 确认网站是否已配置 IndexNow（Check根目录是否有 key 文件）
  2. 生成 IndexNow API key（32 位十六进制字符串）
  3. 将 key 文件放置在网站根目录（如 /abc123.txt）
  4. 测试 API 调用是否返回 200 状态码
  5. Check CMS 或部署流程是否集成了 IndexNow 推送
  6. 验证推送的 URL 列表是否正确

Configuration

# 1. 生成 key 文件
echo "your-api-key-here" > public/your-api-key-here.txt

# 2. 单个 URL 推送
curl "https://api.indexnow.org/indexnow?url=https://example.com/new-page&key=your-api-key-here"

# 3. 批量推送（POST 请求）
curl -X POST "https://api.indexnow.org/indexnow" \
  -H "Content-Type: application/json" \
  -d '{
    "host": "example.com",
    "key": "your-api-key-here",
    "urlList": [
      "https://example.com/page1",
      "https://example.com/page2"
    ]
  }'

CMS Integration

  • WordPress：安装 IndexNow 官方插件
  • Next.js：在 build 后脚本中调用 API
  • Hugo/Jekyll：在 CI/CD 部署后触发推送
  • 自定义 CMS：在内容发布 hook 中集成

Fix Methods

  • 生成 API key 并放置验证文件
  • 在内容发布流程中集成 IndexNow 推送
  • 批量推送已有但未收录的页面
  • 设置自动化：每次部署后自动推送变更 URL

Verification

推送后在 Bing Webmaster Tools 中查看 URL 提交记录。通常 Bing 会在几小时内抓取推送的 URL。Use Sitemap Audit Tool check URL list format.
