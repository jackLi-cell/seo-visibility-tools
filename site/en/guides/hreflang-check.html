Multilingual Website hreflang Configuration Check

Check if hreflang tag configuration is correct to avoid multilingual pages competing for rankings

What is hreflang

hreflang 是一个 HTML 属性，告诉搜索引擎页面有哪些语言/地区版本。正确配置可以避免多语言页面互相竞争排名，确保用户看到正确语言的页面。

Common Causes（hreflang 配置错误）

  • hreflang 标签缺少自引用（每个页面必须包含指向自己的 hreflang）
  • 语言代码格式错误（应Use ISO 639-1，如 zh 而非 chinese）
  • 缺少 x-default 标签
  • hreflang 不是双向的（A 指向 B，但 B 没有指向 A）
  • URL Use相对路径而非绝对路径

Troubleshooting Steps

  1. 查看页面源码中所有 link rel="alternate" hreflang 标签
  2. 确认每个页面都有自引用的 hreflang 标签
  3. 验证语言代码格式正确（zh-CN、en-US、ja 等）
  4. Check所有 hreflang 是否双向对应
  5. 确认存在 x-default 标签指向默认版本
  6. 验证 hreflang 中的 URL 都返回 200 且没有重定向
  7. Check sitemap 中是否也包含了 hreflang 信息

Correct Configuration Example

<!-- 中文页面 (zh-CN) -->
<link rel="alternate" hreflang="zh-CN" href="https://example.com/zh/page" />
<link rel="alternate" hreflang="en" href="https://example.com/en/page" />
<link rel="alternate" hreflang="ja" href="https://example.com/ja/page" />
<link rel="alternate" hreflang="x-default" href="https://example.com/en/page" />

<!-- Sitemap 中的 hreflang -->
<url>
  <loc>https://example.com/zh/page</loc>
  <xhtml:link rel="alternate" hreflang="zh-CN" href="https://example.com/zh/page"/>
  <xhtml:link rel="alternate" hreflang="en" href="https://example.com/en/page"/>
</url>

Fix Methods

  • 为每个语言版本页面添加完整的 hreflang 标签集
  • 确保所有 hreflang 关系是双向的
  • 添加 x-default 指向默认语言版本
  • Use绝对 URL，确保协议和域名一致
  • 在 sitemap 中也声明 hreflang 关系

Verification

Use HTML Readability Check Tool Check hreflang 标签格式。在 GSC 中查看国际定位报告，确认没有 hreflang 错误。
