Which Pages Should Be in Your Sitemap

Sitemap should only include high-quality public pages, do not stuff all files into it

Purpose of Sitemap

Sitemap tells search engines which pages on your site are worth crawling and indexing. It is not a guarantee of indexing, but a discovery mechanism.

Pages That Should Be in Sitemap

  • All public pages you want indexed
  • Pages returning 200 status code
  • Pages with self-referencing canonical
  • Pages without noindex
  • Pages with substantial content

Content That Should Not Be in Sitemap

  • Non-HTML files (use media sitemap for PDFs, images)
  • Duplicate URLs with parameters
  • Pages with noindex set
  • Source URLs of 301/302 redirects
  • URLs returning 4xx/5xx
  • Low-quality tag pages, empty category pages
  • Internal search result pages
  • Pages requiring login

Initial Submission Rules

  1. New sites should submit no more than 1000 URLs initially
  2. Prioritize submitting the most important pages with richest content
  3. Ensure every URL is accessible (200 status code)
  4. Use lastmod to mark the actual last modification time
  5. Monitor index coverage changes in GSC after submission
  6. Gradually add URLs based on indexing results

Sitemap Format Example

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/page</loc>
    <lastmod>2024-01-15</lastmod>
  </url>
</urlset>

Verification

Use Sitemap Audit Tool check your URL 列表。
