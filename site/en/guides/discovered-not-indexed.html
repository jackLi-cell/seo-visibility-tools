How to Troubleshoot Discovered but Not Indexed

Google discovered the URL but has not crawled it yet, usually due to insufficient crawl priority or page quality signals

What is "Discovered - currently not indexed"

Google Search Console showing "Discovered - currently not indexed" means Google knows this URL exists but has not crawled it yet. This usually means Google considers this page not high enough priority.

Common Causes

  • Insufficient site crawl budget, Google prioritizes crawling more important pages
  • Page lacks internal link support, considered unimportant
  • New site or new page, Google has not had time to crawl yet
  • Slow server response, Google reduced crawl frequency
  • Weak page content, Google predicts low value
  • Too many URLs in sitemap, diluting priority

Troubleshooting Steps

  1. Confirm URL status is indeed "Discovered - currently not indexed" in GSC
  2. Check if the page has sufficient internal links (at least 2-3 internal links)
  3. Confirm the page is in the sitemap and the sitemap has been submitted to Google
  4. Check server response time, ensure TTFB is under 500ms
  5. Confirm robots.txt is not blocking this URL
  6. Check if the page content has substantial value
  7. Use GSC URL Inspection tool to request indexing
  8. Wait 1-2 weeks and check status changes again

Fix Methods

  • Add internal links from high-authority pages to this page
  • Streamline sitemap, keep only high-quality pages
  • Improve page content quality and uniqueness
  • Optimize server performance, reduce response time
  • Manually request indexing in GSC

Verification

After fixing, resubmit in GSC URL Inspection tool and observe status changes within 1-2 weeks. You can also confirm indexing by searching site:yoururl.
