Initial HTML Readable Content Checklist

Confirm search engines can read core content in initial HTML on first crawl

Why Initial HTML Matters

When search engines first crawl a page, they primarily read the initial HTML returned by the server. While Google can execute JavaScript, this requires additional crawl resources and time, with no guarantee of complete rendering.

What Initial HTML Must Contain

  • Complete title tag (not dynamically set by JS)
  • meta description tag
  • At least one H1 tag containing the page topic
  • Core page body content (not dependent on JS loading)
  • canonical tag
  • Structured data (JSON-LD)
  • Internal navigation links

JS Rendering Issues

  • SPA without SSR/SSG, initial HTML only has empty div
  • Content loaded asynchronously via AJAX
  • Key content only inserted into DOM after JS execution
  • Using client-side routing, server returns same HTML for all routes

Solutions

  • Use SSR (Server-Side Rendering) or SSG (Static Site Generation)
  • Ensure key content is visible in initial HTML
  • Use frameworks that support SSR like Next.js, Nuxt.js
  • Consider pre-rendering solutions for pure client-side apps

How to Check

# 查看初始 HTML（不执行 JS）
curl -s https://example.com/page | grep -i "h1|title|description"
