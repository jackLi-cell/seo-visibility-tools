Core Web Vitals 对收录的影响

LCP、INP、CLS 三大核心指标如何影响搜索排名和收录优先级

什么是 Core Web Vitals

Core Web Vitals（核心网页指标）是 Google 用来衡量用户体验的三个关键指标：LCP（最大内容绘制）、INP（交互到下一次绘制）和 CLS（累积布局偏移）。这些指标直接影响搜索排名。

常见原因（指标不达标）

  • LCP 过慢：大图片未优化、服务器响应慢、渲染阻塞资源多
  • INP 过高：JavaScript 执行时间长、主线程阻塞、事件处理器复杂
  • CLS 过大：图片/广告没有预留尺寸、动态内容插入、字体加载闪烁
  • 第三方脚本拖慢页面性能
  • 未使用 CDN 或缓存策略不当

排查步骤

  1. 在 GSC 的 Core Web Vitals 报告中查看整站状态
  2. 使用 PageSpeed Insights 测试具体页面得分
  3. 在 Chrome DevTools Performance 面板录制页面加载
  4. 检查 LCP 元素是什么（通常是首屏大图或标题）
  5. 分析 INP 最差的交互是哪个（点击、输入等）
  6. 使用 Layout Shift Debugger 定位 CLS 来源
  7. 检查第三方脚本对性能的影响

达标阈值

指标良好需改进差
LCP≤ 2.5s2.5s - 4.0s> 4.0s
INP≤ 200ms200ms - 500ms> 500ms
CLS≤ 0.10.1 - 0.25> 0.25

修复方法

  • LCP：压缩图片、使用 WebP/AVIF、预加载关键资源、优化 TTFB
  • INP：拆分长任务、使用 Web Worker、延迟非关键 JS
  • CLS：为图片/视频设置 width/height、避免动态插入内容、使用 font-display: swap
  • 启用 CDN 和浏览器缓存
  • 延迟加载非首屏资源

验证方法

修复后使用 PageSpeed Insights 重新测试。注意 CrUX 数据（真实用户数据）需要 28 天才能更新。GSC 报告也会在数据积累后反映改善。
