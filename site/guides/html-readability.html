初始 HTML 可读内容检查清单

确认搜索引擎首次抓取时能在初始 HTML 中读到核心内容

为什么初始 HTML 很重要

搜索引擎首次抓取页面时，主要读取服务器返回的初始 HTML。虽然 Google 可以执行 JavaScript，但这需要额外的抓取资源和时间，且不保证完整渲染。

初始 HTML 必须包含什么

  • 完整的 title 标签（不是 JS 动态设置的）
  • meta description 标签
  • 至少一个 H1 标签包含页面主题
  • 页面核心正文内容（不依赖 JS 加载）
  • canonical 标签
  • 结构化数据（JSON-LD）
  • 内部导航链接

JS 渲染问题

  • SPA 没有 SSR/SSG，初始 HTML 只有空 div
  • 内容通过 AJAX 异步加载
  • 关键内容在 JS 执行后才插入 DOM
  • 使用 client-side routing，服务器对所有路由返回相同 HTML

解决方案

  • 使用 SSR（服务端渲染）或 SSG（静态生成）
  • 确保关键内容在初始 HTML 中可见
  • 使用 Next.js、Nuxt.js 等支持 SSR 的框架
  • 对纯客户端应用考虑预渲染方案

检查方法

# 查看初始 HTML（不执行 JS）
curl -s https://example.com/page | grep -i "h1|title|description"
