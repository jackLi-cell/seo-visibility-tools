Robots.txt 和 Noindex 怎么检查

检查 robots.txt、meta robots 和 X-Robots-Tag 是否误屏蔽页面

Robots.txt 基础

robots.txt 是放在网站根目录的文本文件，告诉爬虫哪些路径可以抓取、哪些不可以。注意：robots.txt 只是建议，不是强制屏蔽。

Robots.txt 语法

# 允许所有爬虫访问所有内容
User-agent: *
Allow: /

# 屏蔽特定目录
User-agent: *
Disallow: /admin/
Disallow: /private/

# 指定 Sitemap 位置
Sitemap: https://example.com/sitemap.xml

Meta Robots 标签

<!-- 不收录此页面 -->
<meta name="robots" content="noindex">

<!-- 不收录且不跟踪链接 -->
<meta name="robots" content="noindex, nofollow">

X-Robots-Tag HTTP 头

X-Robots-Tag: noindex
X-Robots-Tag: noindex, nofollow

常见误屏蔽场景

  • robots.txt 中 Disallow: / 屏蔽了整站
  • 测试环境的 noindex 标签被带到了生产环境
  • CMS 的"阻止搜索引擎索引"选项被勾选
  • CDN 或服务器配置添加了全局 X-Robots-Tag
  • 框架默认给某些路由加了 noindex

排查步骤

  1. 访问 /robots.txt 检查是否有过度屏蔽
  2. 查看页面源码搜索 "noindex"
  3. 用开发者工具检查响应头中的 X-Robots-Tag
  4. 检查 CMS 后台的 SEO 设置
  5. 确认 CDN 配置没有添加额外的 robots 头

验证方法

使用 Robots 检查器 粘贴你的 robots.txt 或 meta 标签进行检查。
