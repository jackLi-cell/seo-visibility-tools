结构化数据和 FAQ 基础检查

JSON-LD 结构化数据的基本检查方法和常见错误

什么是结构化数据

结构化数据是用标准格式（通常是 JSON-LD）向搜索引擎描述页面内容的方式。它帮助搜索引擎理解页面的类型、内容和关系。

常见原因（结构化数据问题）

  • JSON 格式错误（缺少逗号、引号不匹配）
  • 使用了不存在的 schema 类型或属性
  • 结构化数据内容与页面可见内容不一致
  • 缺少必需属性
  • 日期格式不正确（应使用 ISO 8601）

排查步骤

  1. 查看页面源码，找到所有 script type="application/ld+json" 标签
  2. 将 JSON-LD 内容粘贴到 JSON 验证器中检查格式
  3. 使用 Google 富媒体搜索结果测试工具验证 schema 有效性
  4. 对比结构化数据中的内容与页面可见内容是否一致
  5. 检查是否缺少必需属性（如 Article 需要 headline、author）
  6. 确认日期格式为 ISO 8601（如 2024-01-15）
  7. 验证 URL 字段使用绝对路径而非相对路径

JSON-LD 基础

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "页面标题",
  "description": "页面描述",
  "dateModified": "2024-01-15",
  "author": {
    "@type": "Person",
    "name": "作者名"
  }
}
</script>

FAQPage 结构化数据

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "问题1？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "答案1"
      }
    }
  ]
}
</script>

WebApplication 结构化数据

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebApplication",
  "name": "工具名称",
  "description": "工具描述",
  "applicationCategory": "SEO Tool",
  "operatingSystem": "Web Browser",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "USD"
  }
}
</script>

修复方法

  • 使用在线 JSON 验证器修复格式错误
  • 参考 schema.org 官方文档确认属性名称
  • 确保结构化数据与页面可见内容完全一致
  • 添加所有必需属性，补充推荐属性
  • 统一使用 ISO 8601 日期格式

验证方法

使用 HTML 可读性检查工具 检查 JSON-LD 格式是否有效。也可以使用 Google 的富媒体搜索结果测试工具在线验证。
