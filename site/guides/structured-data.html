结构化数据和 FAQ 基础检查

JSON-LD 结构化数据的基本检查方法和常见错误

什么是结构化数据

结构化数据是用标准格式（通常是 JSON-LD）向搜索引擎描述页面内容的方式。它帮助搜索引擎理解页面的类型、内容和关系。

JSON-LD 基础

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "页面标题",
  "description": "页面描述",
  "dateModified": "2024-01-15",
  "author": {
    "@type": "Person",
    "name": "作者名"
  }
}
</script>

FAQPage 结构化数据

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "问题1？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "答案1"
      }
    }
  ]
}
</script>

WebApplication 结构化数据

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebApplication",
  "name": "工具名称",
  "description": "工具描述",
  "applicationCategory": "SEO Tool",
  "operatingSystem": "Web Browser",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "USD"
  }
}
</script>

常见错误

  • JSON 格式错误（缺少逗号、引号不匹配）
  • 使用了不存在的 schema 类型或属性
  • 结构化数据内容与页面可见内容不一致
  • 缺少必需属性
  • 日期格式不正确（应使用 ISO 8601）

验证方法

使用 HTML 可读性检查工具 检查 JSON-LD 格式是否有效。也可以使用 Google 的富媒体搜索结果测试工具在线验证。
